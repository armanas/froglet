## Launch promise

- create a temporary identity with a 15-minute TTL
- run the hosted flow through a shared Froglet gateway
- keep the temporary identity by binding an email address
- stay free-only at launch
- keep payment-rail onboarding on the local path

## Public API contract

The hosted gateway exposes these endpoints:

```text
POST /api/sessions
POST /api/sessions/claim
POST /api/sessions/verify
POST /api/sessions/resume
```

Session-scoped discovery, quote, deal, and execute endpoints sit behind the session created by `POST /api/sessions`.

## User flow

1. `POST /api/sessions` creates a temporary identity.
2. The session can discover services, request a quote, create a deal, and execute a free-only flow.
3. `POST /api/sessions/claim` sends a 6-digit code to the supplied email address.
4. `POST /api/sessions/verify` binds the email to the session and converts it to long-term ownership.
5. `POST /api/sessions/resume` reopens a saved identity with email plus verification code.

## Boundary

This public repo does **not** contain the hosted gateway implementation, its operator controls, or its email and abuse-control stack. Those pieces belong in the separate private repo that serves `try.froglet.dev`.

The public side of that split is documented here so the launch story stays strict and the local repo does not pretend to ship the hosted backend.

## Local path

When you need payment-rail onboarding or full self-host control, use [Run Locally](/learn/quickstart/).

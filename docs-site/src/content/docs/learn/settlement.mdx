## The Lightning Network

The **Lightning Network** is a payment layer built on Bitcoin. It enables instant, low-fee payments by creating payment channels between nodes. Instead of recording every transaction on the blockchain, parties exchange signed IOUs and only settle to the chain when channels open or close.

## Hash Time-Locked Contracts (HTLCs)

Lightning payments use **HTLCs**:

1. The receiver generates a random secret (the **preimage**)
2. The receiver computes $H = \text{SHA-256}(\text{preimage})$ (the **payment hash**)
3. The sender locks funds: "I'll pay you X sats if you reveal the preimage of H within T seconds"
4. The receiver reveals the preimage to claim the funds
5. If the timeout expires, the sender gets their money back

## Froglet's two-leg model

Every paid deal splits into two Lightning payments:

| Leg | Amount | When | Purpose |
|-----|--------|------|---------|
| **Base fee** | Fixed (e.g., 1000 msat) | Locks on admission | Prevents free-riding |
| **Success fee** | Fixed (e.g., 4000 msat) | Settles on success | Fair to requester |

### Why two legs?

- **Requester protection**: If the provider fails, the success fee is never charged
- **Provider protection**: The base fee compensates for reserved resources
- **No escrow needed**: The Lightning HTLCs enforce the rules cryptographically

### Units

```
1 bitcoin    = 100,000,000 satoshis (sats)
1 satoshi    = 1,000 millisatoshis (msat)

Froglet prices in msat.
Typical compute deal: 5,000 msat = 5 sats
```

:::note
Froglet also supports `settlement_method: "none"` for free services. The deal flow still runs — quote, deal, execute, receipt — you just skip the payment.
:::

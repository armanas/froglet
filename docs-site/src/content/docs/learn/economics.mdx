## Definitions

Let $\mathcal{N}$ be the set of all froglet nodes. Each node $n \in \mathcal{N}$ has a secp256k1 keypair $(k_n, P_n)$ where $P_n = k_n \cdot G$.

A **service** $s$ is characterized by:

$$s = (\text{offer\_kind}, \text{runtime}, b_s, f_s, \mathbf{L}_s)$$

where $b_s$ is the base fee, $f_s$ is the success fee, and $\mathbf{L}_s = (t_{\max}, m_{\max}, \phi_{\max})$ is the execution limit vector (time, memory, fuel).

## The deal as a game

A deal between requester $R$ and provider $P$ is a two-player sequential game:

$$\text{Quote} \xrightarrow{\text{P signs}} \text{Deal} \xrightarrow{\text{R signs}} \text{Execute} \xrightarrow{\text{P runs}} \text{Receipt} \xrightarrow{\text{P signs}}$$

### Payoffs

Let $v$ be the value to the requester, $c$ the provider's cost, $p = b_s + f_s$ the total price.

| Outcome | Requester | Provider |
|---------|-----------|----------|
| Success | $v - p$ | $p - c$ |
| Failure | $-b_s$ | $b_s - c$ |
| No deal | $0$ | $0$ |

### Participation constraints

Requester participates if $v \geq b_s + f_s$. Provider participates if $b_s + f_s \geq c$. Gains from trade must be positive: $v - c > 0$.

## Two-leg settlement analysis

Let $q \in [0, 1]$ be the probability the provider successfully executes.

**Requester expected payoff:**

$$\mathbb{E}[\pi_R] = q(v - b_s - f_s) + (1-q)(-b_s) = qv - b_s - qf_s$$

**Provider expected payoff:**

$$\mathbb{E}[\pi_P] = q(b_s + f_s - c) + (1-q)(b_s - c) = b_s + qf_s - c$$

### Incentive compatibility

The provider is incentivized to maximize $q$:

$$\frac{\partial \mathbb{E}[\pi_P]}{\partial q} = f_s > 0$$

The success fee acts as the provider's marginal incentive for reliability. Higher $f_s$ = stronger incentives for quality execution.

### Optimal fee structure

With total price $\bar{p}$ fixed, the equilibrium split for a risk-neutral provider with reliability $q^*$:

$$b_s^* = \frac{c}{1 + q^*}, \quad f_s^* = \bar{p} - b_s^*$$

This ensures the provider breaks even on failed executions while the success fee captures the surplus.

## Network economics

### Marketplace viability

The marketplace charges $(b_M, f_M)$ per search. With query rate $\lambda$ and $N_P$ indexed providers:

$$\text{Rev}_M = \lambda \cdot (b_M + q_M f_M)$$

$$C_M = c_{\text{idx}} \cdot N_P + c_{\text{qry}} \cdot \lambda$$

Viable when $\text{Rev}_M > C_M$.

### Provider discovery value

A provider's expected value from marketplace registration:

$$\Delta V_P = \lambda_M \cdot q_P \cdot (b_P + f_P - c_P) - \text{registration cost}$$

Rational when $\Delta V_P > 0$.

### Network effects

The marketplace exhibits cross-side network effects. Steady-state:

$$N_P^* = g_P(N_R^*), \quad N_R^* = g_R(N_P^*)$$

where $g_P$ and $g_R$ are increasing. Two equilibria: trivial $(0, 0)$ and a non-trivial fixed point where both sides participate.

## Welfare analysis

Total welfare:

$$W = \sum_{\text{deals}} (v_i - c_i) - \text{transaction costs}$$

The protocol maximizes $W$ by:

1. **Minimizing information asymmetry** — signed receipts make quality observable
2. **Eliminating counterparty risk** — HTLCs remove the need for trust
3. **Reducing search costs** — marketplace aggregates information
4. **Enabling price discovery** — competition drives prices toward cost

In the limit of a large market:

$$\lim_{|\mathcal{N}| \to \infty} p_s \to c_s + \epsilon$$

Perfect competition drives prices toward cost, with the marketplace extracting the remaining friction as query fees.

:::tip[The economic conclusion]
Froglet creates a market where trust is computable from signed evidence, incentives are enforced by cryptography, and the equilibrium price reflects the true cost of computation plus the provider's reliability premium.
:::

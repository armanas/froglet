## TL;DR — one copy-paste block

If you already have Docker installed and just want the local stack up, this
single block goes from zero to running Froglet with a Claude Code MCP config
and Lightning mock settlement:

```bash
curl -fsSL https://raw.githubusercontent.com/armanas/froglet/main/scripts/install.sh | sh
./scripts/setup-agent.sh --target claude-code
./scripts/setup-payment.sh lightning
set -a && . ./.froglet/payment/lightning.env && export FROGLET_HOST_READABLE_CONTROL_TOKEN=true && set +a && docker compose up --build -d
```

The rest of this page walks the four steps individually if you want to swap
the agent target, change the payment rail, or understand what each step
actually does.

## Prerequisites

- **Linux x86_64 / arm64** or **macOS arm64**
- **`curl`** to download the installer
- **Node.js 18+** if you want the local MCP server for Claude Code or Codex
- Optional: **Docker** if you want the local provider/runtime stack or the published images
- Optional: a configured **GCP project** if you want the single-VM self-host wrapper

## 1. Install Froglet

Install the latest tagged release into `~/.local/bin`:

```bash
curl -fsSL https://raw.githubusercontent.com/armanas/froglet/main/scripts/install.sh | sh
```

Pin a version or change the install dir if needed:

```bash
curl -fsSL https://raw.githubusercontent.com/armanas/froglet/main/scripts/install.sh | VERSION=v0.1.0 sh
```

```bash
curl -fsSL https://raw.githubusercontent.com/armanas/froglet/main/scripts/install.sh | INSTALL_DIR=/usr/local/bin sh
```

By default the installer fetches `froglet-node`, verifies it against `SHA256SUMS`, and installs it locally. `froglet-node` is the public single-binary path. The release pipeline also publishes provider, runtime, and MCP Docker images.

## 2. Connect an agent

Generate the exact local config file for the agent you plan to use:

```bash
./scripts/setup-agent.sh --target claude-code
```

```bash
./scripts/setup-agent.sh --target codex
```

```bash
./scripts/setup-agent.sh --target openclaw
```

The helper writes a ready-to-use file instead of asking you to hand-edit JSON or TOML:

| Target | Default output |
|-------|----------------|
| `claude-code` | `.mcp.json` |
| `codex` | `.codex/config.toml` |
| `openclaw` | `.froglet/openclaw.config.json` |

The generated files default to:

- provider URL `http://127.0.0.1:8080`
- runtime URL `http://127.0.0.1:8081`
- provider token path `<repo-root>/data/runtime/froglet-control.token`
- runtime token path `<repo-root>/data/runtime/auth.token`

If you plan to use the generated config against Docker Compose, start Compose
with `FROGLET_HOST_READABLE_CONTROL_TOKEN=true` so `<repo-root>/data/runtime/froglet-control.token`
is readable on the host. The single-binary path does not need this opt-in.

More detail: [Connect Agents](/learn/agents/)

## 3. Connect a payment rail

Generate a validated env snippet for the rail you want to launch with:

```bash
./scripts/setup-payment.sh lightning
```

That path uses Lightning mock mode by default, which is enough to exercise the paid-deal flow locally without a wallet.

For the other launch rails:

```bash
FROGLET_STRIPE_SECRET_KEY=sk_test_... ./scripts/setup-payment.sh stripe
```

```bash
FROGLET_X402_WALLET_ADDRESS=0x... ./scripts/setup-payment.sh x402
```

Each run writes an env snippet, prints the exact inputs still required, and runs a verification probe for that rail.

More detail: [Payment Rails](/learn/payment-rails/)

## 4. Start the local stack

Load the payment snippet you generated and start the default local topology:

```bash
set -a
. ./.froglet/payment/lightning.env
set +a
export FROGLET_HOST_READABLE_CONTROL_TOKEN=true
docker compose up --build -d
```

Replace `lightning.env` with `stripe.env` or `x402.env` when that is the rail
you configured.

That starts provider on `127.0.0.1:8080` and runtime on `127.0.0.1:8081`.

That opt-in is required only when host-side agents read the provider control
token from `./data/runtime/froglet-control.token`. If you want the single-binary
path instead of Docker Compose, start `froglet-node` directly after loading the
same env snippet.

For a single GCP VM, the public wrapper keeps a stable interface:

```bash
FROGLET_GCP_PROJECT=your-project ./scripts/deploy_gcp_single_vm.sh create
FROGLET_GCP_PROJECT=your-project ./scripts/deploy_gcp_single_vm.sh deploy
FROGLET_GCP_PROJECT=your-project ./scripts/deploy_gcp_single_vm.sh status
```

## 5. Check the local control plane

Confirm the provider and runtime are healthy:

```bash
curl http://127.0.0.1:8080/health
curl http://127.0.0.1:8081/health
```

Inspect the provider identity:

```bash
curl http://127.0.0.1:8080/v1/node/identity
```

## 6. Execute a WASM workload

Submit a WASM binary for execution:

```bash
WASM_HEX=0061736d01000000010c0260017f017f60027f7f017e03030200010503010001071803066d656d6f7279020005616c6c6f6300000372756e00010a0b02040041100b040042020b0b08010041000b023432

curl -X POST http://127.0.0.1:8081/v1/node/execute/wasm \
  -H "Content-Type: application/json" \
  -d "{\"wasm_hex\":\"$WASM_HEX\",\"input\":{}}"
```

This local runtime helper executes the workload and returns the result plus a
local payment receipt when one is used. The signed quote/deal flow is separate
and uses `/v1/provider/quotes` plus `/v1/provider/deals`.

## 7. Requester side

If you started in dual mode (the default), the requester API is already running on port 8081. If you started as provider-only, start a second node for the requester:

```bash
FROGLET_NODE_ROLE=runtime \
FROGLET_RUNTIME_LISTEN_ADDR=127.0.0.1:8081 \
FROGLET_PAYMENT_BACKEND=none \
froglet-node
```

The requester handles deal negotiation, payment settlement, and receipt verification.

## 8. Register with a marketplace (optional)

To make your provider discoverable:

```bash
FROGLET_PAYMENT_BACKEND=none \
FROGLET_LISTEN_ADDR=127.0.0.1:8080 \
FROGLET_MARKETPLACE_URL=https://mkt.froglet.dev \
froglet-node
```

The provider automatically registers its signed descriptor and offers with the
configured external marketplace on startup. A default marketplace exists, but
its implementation lives outside this public repo.

## 9. Install from source instead (optional)

If you are developing on Froglet itself or want to build locally:

```bash
git clone https://github.com/armanas/froglet.git
cd froglet
cargo build --release
```

## 10. Stake your identity

After registering, stake non-refundable value into your identity to build trust. Requesters use your stake to decide if dealing with you is safe.

```bash
curl -X POST http://127.0.0.1:8080/v1/provider/deals \
  -H "Content-Type: application/json" \
  -d '{
    "offer_id": "marketplace.stake",
    "input": { "provider_id": "YOUR_NODE_ID", "amount_msat": 10000 }
  }'
```

The marketplace records a non-refundable deposit. Your stake T = total_staked_msat is the complete trust signal. Requesters check: is `stake / deal_value` large enough? If your stake is 10,000 msat and the deal costs 8 msat, the safety ratio is 1,250x — cheating would be irrational.

Top up your stake over time with `marketplace.topup`. See [Economics](/learn/economics/) for the full staked reputation model.

## What just happened

Every interaction produced signed artifacts:

1. **Descriptor** — your provider declared its identity and capabilities
2. **Offer** — it published what it can execute and at what price
3. **Quote** — it priced a specific workload
4. **Deal** — the requester committed
5. **Receipt** — the provider signed the outcome

Each artifact references the previous by SHA-256 hash. Tamper with any link and verification fails.

## Environment variable reference

| Variable | Default | Description |
|----------|---------|-------------|
| `FROGLET_NODE_ROLE` | `dual` | `dual`, `provider`, or `runtime` |
| `FROGLET_LISTEN_ADDR` | `127.0.0.1:8080` | Provider API bind address |
| `FROGLET_RUNTIME_LISTEN_ADDR` | `127.0.0.1:8081` | Runtime API bind address |
| `FROGLET_PAYMENT_BACKEND` | `none` | `none`, `lightning`, `stripe`, or `x402` |
| `FROGLET_LIGHTNING_MODE` | `mock` | `mock` or `lnd_rest` |
| `FROGLET_STRIPE_SECRET_KEY` | — | Stripe test secret key for `stripe` |
| `FROGLET_X402_WALLET_ADDRESS` | — | Wallet address for `x402` |
| `FROGLET_X402_NETWORK` | `base` | x402 network selection (`base` only) |
| `FROGLET_NETWORK_MODE` | `clearnet` | `clearnet`, `tor`, or `dual` |
| `FROGLET_MARKETPLACE_URL` | — | Auto-register with marketplace |
| `FROGLET_DATA_DIR` | `./data` | Data directory for SQLite and keys |
| `FROGLET_IDENTITY_AUTO_GENERATE` | `true` | Generate identity on first run |
